Deploy with your AI agent

Copy this prompt, paste it into Claude Code, ChatGPT, Cursor, or any AI coding assistant. Answer a few questions. Your project goes live.

Compatible with: Claude Code, ChatGPT, Cursor, Windsurf, Any AI assistant with terminal access

Deploy Prompt 
You are helping me deploy my project to usectl — a managed deployment platform that handles everything automatically: building my app, provisioning databases, setting up file storage, and giving me a live HTTPS URL.

Here is what I need you to do:

**Step 1 — Install the usectl CLI**
Run the following command to install usectl on my machine:
  curl -fsSL https://usectl.com/install.sh | bash

If it needs sudo, tell me to run it manually with sudo. Then verify:
  usectl version

**Step 2 — Log in**
Log me into my usectl account:
  usectl login

If I don't have an account yet, tell me to sign up at usectl.com first.

**Step 3 — Connect GitHub**
Run: usectl github login
This opens a browser to authorize the GitHub App. After I confirm, check:
  usectl github installations

If the repo I want to deploy isn't listed, tell me to add it at the GitHub App settings page.

**Step 4 — Gather project info**
Ask me:
  1. What is your GitHub repository URL?
  2. What port does your app listen on? (default: 3000)
  3. Does your project need a PostgreSQL database? (yes/no)
  4. Does your project need file storage (S3-compatible)? (yes/no)
  5. Do you want a custom domain? (yes/no — if yes, what domain?)

**Step 5 — Create the project**
Using my answers:
  usectl projects create \
    --name <SHORT_NAME> \
    --repo <GITHUB_URL> \
    --domain <SHORT_NAME> \
    --port <PORT> \
    [--database] \
    [--storage]

**Step 6 — Deploy**
Trigger deployment:
  usectl projects deploy <PROJECT_ID>

Check status until it's running:
  usectl projects status <PROJECT_ID>

**Step 7 — Custom domain (if needed)**
  usectl domains add --project <PROJECT_ID> --domain <DOMAIN>
Tell me what DNS record to add.

**Step 8 — Confirm**
Show me the live URL and confirm the project status.

Throughout this process:
- Explain each step before running commands
- If any command fails, show the error and suggest a fix
- If the GitHub App needs configuration, give me the exact URL to visit
- Keep things simple
