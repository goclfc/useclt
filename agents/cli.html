usectl CLI Reference

Every command, every flag. Optimized for AI agent consumption.

Install

Install the usectl CLI and verify it is available on your PATH.

bash 
curl -fsSL https://usectl.com/install.sh | bash
usectl version

Authentication

Log in to usectl and authorize GitHub access so the platform can clone your repositories.

bash 
usectl login              # Opens browser for OAuth login
usectl github login       # Authorizes GitHub App access

Projects

Create, manage, deploy, and monitor your projects.

bash 
usectl projects list [--json]
usectl projects create --name <name> --repo <github-url> --domain <subdomain> --port <port> [--database] [--storage]
usectl projects get <project-id> [--json]
usectl projects status <project-id>
usectl projects deploy <project-id>
usectl projects build-logs <project-id> <deployment-id>
usectl projects logs <project-id> [--follow]
usectl projects delete <project-id>

GitHub

Manage GitHub App installations and browse accessible repositories.

bash 
usectl github installations [--json]
usectl github repos --installation <installation-id> [--json]
usectl github app-info [--json]

Domains

Add, list, and remove custom domains for your projects. SSL is provisioned automatically.

bash 
usectl domains add --project <project-id> --domain <domain>
usectl domains list --project <project-id>
usectl domains remove --project <project-id> --domain <domain>

Environment Variables

Set, list, and remove environment variables. Changes take effect on the next deploy.

bash 
usectl env set --project <project-id> KEY=value KEY2=value2
usectl env list --project <project-id>
usectl env remove --project <project-id> KEY

Common Patterns for Agents

A full deploy flow an AI agent can follow from start to finish.

bash — full deploy flow 
# 1. Install CLI
curl -fsSL https://usectl.com/install.sh | bash

# 2. Authenticate
usectl login
usectl github login

# 3. Find the GitHub installation and repo
usectl github installations --json
usectl github repos --installation <installation-id> --json

# 4. Create the project
usectl projects create \
  --name my-app \
  --repo https://github.com/user/my-app \
  --domain my-app \
  --port 3000 \
  --database \
  --storage

# 5. Set environment variables
usectl env set --project <project-id> NODE_ENV=production SECRET_KEY=abc123

# 6. Deploy
usectl projects deploy <project-id>

# 7. Check status and logs
usectl projects status <project-id>
usectl projects logs <project-id> --follow

Error Handling

Common errors agents encounter and how to resolve them.

Error Meaning Agent Action
"commit_hash is required" GitHub App can't access the repo Tell user to add the repo to the GitHub App installation
"required flag(s) 'installation' not set" Need installation ID first Run usectl github installations first
"API error 502" Temporary server issue Retry after 5 seconds
"sudo: a terminal is required" CLI install needs sudo Tell user to run the install command manually
