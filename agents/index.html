usectl for AI Agents

usectl is designed to be used by AI agents. There are four ways to integrate.

1. CLI — Works with every agent today

The usectl CLI is a Go binary that runs in any terminal. Every AI coding agent — Claude Code, Cursor, Windsurf, ChatGPT — already knows how to run CLI commands. No integration needed.

Install 
curl -fsSL https://usectl.com/install.sh | bash

Best for: Any agent with terminal access. Universal. Works today.

CLI Reference →

2. MCP Connector — Tighter integration

The usectl MCP server exposes deploy, logs, status, and project management as tool calls inside MCP-compatible AI clients — Claude Code, VS Code, Cursor, Cowork.

Setup for Claude Code 
claude mcp add usectl npx @usectl/mcp-server --env USECTL_API_KEY=<key>

Best for: Claude Code, Cursor, and other MCP clients. Richer context. Structured tool calls.

MCP Documentation →

3. Skill — Pre-built workflow

The usectl skill is a pre-built workflow that guides the agent through the entire deploy process — from installing the CLI to verifying the project is live. No prompt engineering needed.

Install in Claude Code 
claude install usectl

Best for: Claude Code / Cowork users who want a one-click deploy experience.

Skill Documentation →

4. Context Doc — Paste into any agent

A plain-text document you paste into any AI conversation to give the agent full context about usectl — what it is, how the CLI works, and how to deploy.

Best for: ChatGPT, Gemini, or any agent that doesn't support MCP or skills. Works everywhere.
