# usectl — Agent Context

You are helping the user deploy their project using usectl, a managed deployment platform. usectl provides: server hosting, PostgreSQL database, S3-compatible storage (MinIO), SSL/TLS, custom domains, CI/CD from GitHub — all in one platform for $39/month per project.

## CLI Installation
curl -fsSL https://usectl.com/install.sh | bash
(Needs sudo on macOS. If it fails, ask user to run with sudo.)

## Authentication
usectl login                    # OAuth login via browser
usectl github login             # Authorize GitHub App (one-time)

## Core Workflow

### 1. Create a project
usectl projects create \
  --name <name> \
  --repo <github-repo-url> \
  --domain <subdomain> \
  --port <port> \
  [--database] \
  [--storage]

### 2. Deploy
usectl projects deploy <project-id>

### 3. Check status
usectl projects status <project-id>

### 4. View logs
usectl projects logs <project-id> --follow

## Important Notes
- The GitHub App must have access to the repo before deploying. If you get "commit_hash is required (could not auto-resolve HEAD)", the repo is not accessible. Run `usectl github installations` to find the installation ID, then tell the user to add the repo at: https://github.com/settings/installations/<installation-id>
- Auto-Dockerfile detection supports Next.js, Node.js, and static sites. For other frameworks, the user needs a Dockerfile in the repo root.
- Projects are live at https://<domain>.usectl.app
- Every git push to main auto-rebuilds and redeploys.

## Available Commands

usectl projects get <id> [--json]

usectl projects logs <id> [--follow]
usectl projects delete <id>
usectl github installations [--json]
usectl github repos --installation <id> [--json]
usectl domains add --project <id> --domain <domain>
usectl domains list --project <id>
usectl env set --project <id> KEY=value
usectl env list --project <id>
