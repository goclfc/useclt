Deploy your first project

From zero to a live HTTPS URL in under 5 minutes. Your AI agent can do this for you — or follow the steps manually.

Quickest path: If you use Claude Code, Cursor, or ChatGPT — just paste the deploy prompt and let your agent handle everything below.

Step 1 — Install the CLI

Run the install script to download and set up the usectl binary:

bash 
curl -fsSL https://usectl.com/install.sh | bash

On macOS this requires sudo. If prompted, run with sudo bash instead.

Verify the installation:

bash 
usectl version

Step 2 — Log in

Authenticate with your usectl account:

bash 
usectl login

This opens your browser for OAuth. Once approved, the CLI stores your credentials locally.

Step 3 — Connect GitHub

Authorize the usectl GitHub App so the platform can access your repositories:

bash 
usectl github login

This is a one-time setup. The GitHub App is installed on your account and grants access to the repos you select.

To add more repos later, find your installation ID and configure it on GitHub:

bash 
usectl github installations

Then go to https://github.com/settings/installations/<ID> to manage repository access.

Step 4 — Create your project

Create a new project linked to your GitHub repository:

bash 
usectl projects create \
  --name my-app \
  --repo https://github.com/yourname/your-repo.git \
  --domain my-app \
  --port 3000

Required flags:

  • --name — A unique name for your project
  • --repo — The full GitHub repository URL
  • --domain — The subdomain for your .usectl.app URL
  • --port — The port your application listens on

Optional flags:

  • --database — Provision a PostgreSQL database
  • --storage — Provision S3-compatible object storage

Step 5 — Deploy

Trigger a deployment:

bash 
usectl projects deploy <project-id>

Watch the build logs in real time:

bash 
usectl projects build-logs <project-id> <deployment-id>

Check the status of your project:

bash 
usectl projects status <project-id>

Step 6 — You're live

Your project is now running at:

URL 
https://my-app.usectl.app

SSL is provisioned automatically. Push-to-deploy is active — every push to your default branch triggers a new deployment.

Optional — Custom domain

Point your own domain at your project:

bash 
usectl domains add --project <project-id> --domain yourdomain.com

TLS certificates are provisioned automatically via Let's Encrypt. See the Domains guide for DNS configuration details.

Troubleshooting

Problem Fix
"commit_hash is required" GitHub App doesn't have access to the repo. Add it at GitHub Settings → Applications → usectl → Configure.
Install script needs sudo Run: curl -fsSL https://usectl.com/install.sh | sudo bash
Build fails with no Dockerfile Add a Dockerfile to your repo root. Auto-detection supports Next.js, Node.js, and static sites.
502 on github installations Re-run usectl github login to refresh your GitHub authorization.

For more help, see the full Troubleshooting guide or ask in Discord.
