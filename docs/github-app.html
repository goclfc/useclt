GitHub App Setup

The GitHub App is how usectl accesses your repositories to clone code and trigger builds. This is the #1 friction point in the deploy flow — this guide covers setup and every known issue.

What the GitHub App Does

When you run usectl projects deploy, the platform needs to clone your repository. The GitHub App provides secure, scoped access to your repos without requiring personal access tokens.

The app requests read-only access to repository contents and metadata. It does not modify your code, create branches, or push commits.

First-Time Setup

Run the GitHub OAuth flow to link your GitHub account:

bash 
usectl github login

This opens your browser to authorize the usectl GitHub App. After authorizing:

  1. Select which GitHub account or organization to install the app on
  2. Choose "All repositories" or select specific repos
  3. Click Install

Verify the installation:

bash 
usectl github installations --json

You should see your account/org with an installation ID.

Adding a New Repository

If you created a new repo after installing the GitHub App, you need to add it to the installation:

  1. Find your installation ID:
    bash 
    usectl github installations
  2. Go to: https://github.com/settings/installations/<INSTALLATION_ID>
  3. Under "Repository access", click "Select repositories" and add the new repo
  4. Click Save

Verifying Repo Access

List repos accessible by an installation:

bash 
usectl github repos --installation <INSTALLATION_ID> --json

If your repo doesn't appear in the list, the GitHub App can't access it. Add it via the installation settings link above.

Common Issues

"commit_hash is required (could not auto-resolve HEAD)"

This is the most common error. It means the GitHub App cannot access the repository.

Fix:

  1. Run usectl github installations to get your installation ID
  2. Go to https://github.com/settings/installations/<ID>
  3. Add the repository to the GitHub App
  4. If the project was created before the app had access, delete and recreate it:
    bash 
    usectl projects delete <project-id>
usectl projects create --name my-app --repo https://github.com/user/repo.git --domain my-app --port 3000

"API error 502" on github installations

Your GitHub token may be expired or invalid.

Fix: Re-authenticate with usectl github login

Project created but can't deploy

If you created a project before connecting the GitHub App, the platform can't auto-detect the installation. Delete the project and recreate it after the GitHub App is properly configured.

Organization repositories

For org repos, the GitHub App must be installed on the organization, not just your personal account. An org admin may need to approve the installation.

How It Works Internally

When you create a project with --repo, usectl checks all your GitHub App installations to find one that has access to that repository. The installation ID is stored with the project.

On deploy, usectl uses the installation's credentials to clone the repo at the latest commit on your configured branch (default: main).
