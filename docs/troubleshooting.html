Troubleshooting

Common errors and fixes, organized by category. Written so both humans and AI agents can quickly find solutions.

Authentication

ErrorCauseFix
not logged in or unauthorized CLI session expired or not logged in Run usectl login
token expired Auth token has expired Run usectl login to re-authenticate

GitHub App

ErrorCauseFix
commit_hash is required (could not auto-resolve HEAD) GitHub App can't access the repository Go to https://github.com/settings/installations/<ID> and add the repo. If the project was created before connecting the app, delete and recreate it.
API error 502 on github installations GitHub token expired or invalid Run usectl github login to refresh the token
required flag(s) "installation" not set Command requires an installation ID Run usectl github installations --json first to get the ID
Repo not showing in github repos Repo not added to GitHub App installation Visit GitHub App settings and add the repository

See the full GitHub App guide for detailed setup instructions.

CLI Installation

ErrorCauseFix
sudo: a terminal is required to read the password Install script needs sudo but can't prompt for password Run manually: curl -fsSL https://usectl.com/install.sh | sudo bash
usectl: command not found Binary not in PATH Check /usr/local/bin/usectl exists. If installed elsewhere, add the directory to your PATH.

Build & Deploy

ErrorCauseFix
Build fails — no Dockerfile found usectl couldn't auto-detect the framework Add a Dockerfile to your repo root. Auto-detection supports Next.js, Node.js, and static sites only.
Build succeeds but app crashes App crashes on startup (wrong port, missing env vars, etc.) Check logs: usectl projects logs <id>. Verify --port matches what your app listens on.
Status: stopped (replicas: 0) App was stopped or crashed repeatedly Check logs for crash reason. Fix and redeploy, or run usectl projects start <id>
Deploy triggered but nothing happens Build may be queued or failed silently Check build logs: usectl projects build-logs <project-id> <deployment-id>. Get deployment ID from usectl projects get <id> --json

Domains & SSL

ErrorCauseFix
Custom domain not working DNS not configured or not propagated Add a CNAME record pointing your domain to usectl.app. DNS propagation can take up to 48 hours.
SSL certificate not issued DNS not resolving to usectl yet Verify DNS with dig yourdomain.com CNAME. Certificate auto-provisions once DNS resolves.

Database

ErrorCauseFix
App can't connect to database DATABASE_URL not set or database not provisioned Verify database was enabled: usectl projects get <id> --json. Check env vars: usectl env list --project <id>

Storage

ErrorCauseFix
S3 upload fails Storage not provisioned or credentials wrong Verify storage was enabled. Check S3_ENDPOINT, S3_ACCESS_KEY, S3_SECRET_KEY in env vars.
